# QuickCheck QA — 接单前安全检查清单 (Pre-Gig Checklist)

> 用法:每接到一个新请求,从上往下走一遍。任何一条 ❌ 红线被触发,直接礼貌拒单。
> 原则:危险的不是"看"网站,是"下载、运行、输入"。

---

## 第一关:客户与需求审查(收到邮件时)

- [ ] 客户有具体、合理的需求描述(知道自己网站是干什么的、担心什么)
- [ ] 网址是完整的正常域名(不是缩短链接 bit.ly/tinyurl、不是裸 IP 地址、不是一长串乱码参数)
- [ ] 网站内容和客户的自我描述对得上(说是奥克兰餐厅,打开真的是餐厅)
- [ ] 客户邮箱看起来正常(企业域名邮箱最佳;免费邮箱可以接受,但结合其他信号判断)
- [ ] 没有催单压力("必须今天测完"是社工攻击的经典话术,正常客户不会这样)

### ❌ 红线 — 出现任何一条,直接拒单

- 要求**下载、安装或运行**任何文件、软件、浏览器扩展
- 索要或主动提供**真实账号密码**(测试只用一次性测试账号)
- 让你测试**明显不属于他的网站**(竞争对手的站、"朋友的站"但说不清授权)
- 涉及灰产内容:赌博、成人、仿冒名牌、来路不明的加密货币项目
- 付款方式异常:多付要求退差价(经典 overpayment 诈骗)、用礼品卡/加密货币支付
- 链接是缩短网址且对方拒绝提供原始链接

> 拒单话术(中):"抱歉,这个请求不符合我的服务条款,这单我不能接。"
> 拒单话术(英):"Sorry, this request falls outside my service terms, so I'll have to pass."
> 不需要解释具体哪条——解释得越多,对方越知道下次怎么绕。

---

## 第二关:链接技术预检(打开网站之前)

- [ ] **VirusTotal 扫描**:virustotal.com → URL 标签 → 粘贴网址 → 0 检出才继续(1-2 个小众引擎误报可人工判断,3 个以上检出直接拒)
- [ ] **域名背景检查**:`whois 域名` 或 dnc.org.nz(.nz 域名)查注册时间 — 上周才注册的"老牌生意"网站是危险信号
- [ ] **HTTPS 检查**:浏览器地址栏锁图标正常,证书没有告警(证书报错的站不进)
- [ ] **Google 安全浏览**:transparencyreport.google.com/safe-browsing/search 查一下网址状态

---

## 第三关:环境准备(开始测试之前)

- [ ] 在**隔离环境**中打开:UTM 虚拟机(最安全)或 Mac 上专用的标准用户账户(最低要求)
- [ ] macOS 和浏览器已是最新版本(系统设置 → 通用 → 软件更新)
- [ ] 当前浏览器 profile 里**没有登录**任何个人账户(Gmail、网银、券商一个都不能有)
- [ ] 如需测登录流程:已让客户创建一次性测试账号,并约定测完即删
- [ ] Time Machine 备份在最近 7 天内跑过(万一出事能回滚)

---

## 第四关:范围确认(和客户书面确认)

- [ ] 确认套餐:Smoke ($5) / Deep ($15) / Pre-Launch ($30)
- [ ] 确认测试范围:哪些页面、哪条用户流程
- [ ] **真实交易红线**:明确约定是否允许走到真实付款/真实订单那一步(默认:不允许,测到付款页前一步为止)
- [ ] 交付时间已确认(48 或 72 小时),邮件留底

---

## 测试中守则(贴在显示器上的版本)

1. 只看、只点、只输入测试数据 — 不下载、不安装、不输真实密码
2. 弹出"需要更新 Flash / 安装插件 / 下载查看器" = 立即停止,这本身就是一条 SEV 1 发现(或者这就是个恶意网站)
3. 表单里只填编造的测试数据,永远不填自己的真实姓名、电话、地址
4. 任何不可逆操作(删除、付款、发送)前,对照第四关的约定

---

## 交付后(收尾)

- [ ] 报告已发送,客户已确认收到
- [ ] 提醒客户删除测试账号
- [ ] 收款已到账,截图/记录存档(IRD 报税:这是 sole trader 收入,记入当年所得)
- [ ] 测试用的浏览器 profile / 虚拟机快照已清理或回滚
- [ ] (可选)询问客户是否愿意留一句评价 — 攒够 3 条真实评价就可以上网站了
